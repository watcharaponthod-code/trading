# AlgoTrade — Automated Trading Dashboard Setup Guide

## Overview
This is a comprehensive automated trading management application with Alpaca API integration, real-time market data, live trading charts, and quantitative strategy execution.

## Features Implemented

### 1. **Real-Time Data Integration**
- Live account equity, buying power, and positions from Alpaca
- Real-time market data (OHLCV bars) for watchlist symbols
- Auto-refresh every 15 seconds

### 2. **Database Backend (Neon PostgreSQL)**
Four tables for comprehensive trading history:
- **trades**: Records all executed trades with order IDs, fills, P&L
- **strategy_configs**: Stores 4 default strategies with configurable parameters
- **trade_signals**: Records all signals generated by strategies
- **portfolio_snapshots**: Daily equity/P&L snapshots

### 3. **Dashboard Components**
- **Left Sidebar**: Live watchlist (SPY, QQQ, AAPL, MSFT, TSLA, NVDA, AMD, GOOGL, GLD, SLV)
- **Center Panel**: 
  - Stats cards (Equity, Buying Power, Positions, P&L)
  - Portfolio equity area chart
  - Daily P&L bar chart
  - Price chart with timeframe selection (1M, 5M, 15M, 1H, 1D)
  - Positions and Orders tabs
- **Right Sidebar**: 
  - Strategy selector and auto-run controls
  - Manual order form (market/limit)

### 4. **Strategy Engine**
Four quantitative trading strategies:
1. **Statistical Arbitrage**: Z-score spread trading between SPY/QQQ
2. **Mean Reversion**: Bollinger Bands + RSI on AAPL/MSFT/GOOGL
3. **EMA Momentum**: EMA crossover on TSLA/NVDA/AMD
4. **Pairs Trading**: Long/short pairs on GLD/SLV

Each strategy supports:
- Dry-run mode (analyze without executing)
- Live execution mode
- Auto-run (minute-by-minute signals)
- Signal persistence to database

### 5. **API Routes**
- `/api/account` - Fetch account data
- `/api/positions` - Get open positions
- `/api/orders` - Fetch order history
- `/api/market` - Real-time and historical OHLCV bars
- `/api/trade` - Submit market/limit orders
- `/api/strategy` - Run strategy and execute signals
- `/api/db-strategies` - Get strategy configs from DB
- `/api/db-trades` - Fetch trade history
- `/api/db-signals` - Fetch trade signals
- `/api/db-snapshots` - Fetch portfolio snapshots

## Setup Instructions

### 1. Environment Variables
Add these to your Vercel project Vars:
```
ALPACA_API_KEY=your_key_here
ALPACA_API_SECRET=your_secret_here
ALPACA_BASE_URL=https://paper-api.alpaca.markets/v2
DATABASE_URL=your_neon_connection_string
```

### 2. Database Migration
The migration runs automatically when the app starts, creating:
- `trades` table
- `strategy_configs` table (pre-populated with 4 default strategies)
- `portfolio_snapshots` table
- `trade_signals` table

### 3. Dependencies
All required packages are in `package.json`:
- `@neondatabase/serverless` - Neon PostgreSQL
- `recharts` - Chart components
- `date-fns` - Date formatting
- `lucide-react` - Icons
- Standard shadcn/ui components

## How to Use

### Trading Dashboard
1. **Monitor**: Watch live prices in the left sidebar
2. **Select Symbol**: Click a symbol to view detailed price chart
3. **Strategy**: Choose a strategy in the right panel
4. **Dry Run**: Test the strategy without executing
5. **Live Deploy**: Execute real trades based on signals
6. **Auto-Run**: Enable automatic strategy execution
7. **Manual Trading**: Use the order form for manual trades

### Manual Orders
- Select symbol in main area
- Enter quantity
- Choose Market or Limit type
- Set limit price if needed
- Click Buy or Sell

### View History
- **Positions Tab**: All open positions with P&L
- **Orders Tab**: Last 20 orders with status
- **Portfolio Chart**: Equity over time
- **P&L Chart**: Daily profit/loss

## Data Flow

```
User Input
    ↓
API Route (/api/trade, /api/strategy, etc)
    ↓
Alpaca API (live execution)
    ↓
Database (Neon) ← Persistence
    ↓
Dashboard (real-time updates)
```

## Security Notes

1. **API Keys**: Stored in environment variables, never exposed to client
2. **Server-Side Execution**: All trades execute on the server with verified keys
3. **Paper Trading**: Using Alpaca paper trading endpoint by default
4. **HTTPS Only**: All requests to Alpaca are HTTPS

## Customization

### Add New Strategy
Edit `/lib/strategy.ts`:
1. Add strategy to `DEFAULT_STRATEGIES` array
2. Implement signal generation logic
3. Add to database seed in migration

### Change Watchlist
Edit `/app/page.tsx`:
```javascript
const WATCHLIST_SYMBOLS = ["SPY", "QQQ", ...] // Add/remove symbols
```

### Adjust Refresh Rate
Edit `/app/page.tsx`:
```javascript
const REFRESH_INTERVAL = 15000 // milliseconds
```

## Troubleshooting

### No Data Loading
- Check environment variables are set correctly
- Verify Alpaca API credentials
- Check database connection in Neon

### Orders Not Executing
- Ensure paper trading is enabled
- Check account has sufficient buying power
- Review order validation in `/api/trade`

### Charts Not Rendering
- Check console for fetch errors
- Verify `/api/market` endpoint is working
- Ensure sufficient historical data available

## Production Deployment

1. Switch `ALPACA_BASE_URL` to live trading endpoint
2. Enable additional safeguards (position limits, max losses)
3. Set up monitoring and alerts
4. Add rate limiting to API routes
5. Use separate database credentials
